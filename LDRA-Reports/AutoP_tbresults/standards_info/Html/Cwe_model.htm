Compliance Matrix

A compliance matrix for the CWE standards model is available in the Microsoft Excel XML file standards_info/compliance/cwe_compliance.xml, which may be found relative to the LDRA tool suite installation directory.

Many CWE weaknesses are undecidable. Such weaknesses will be marked as Fully Implemented if all statically checkable aspects of the weakness are addressed. The CWE document precedes each entry with one of the following keywords:

WEAKNESS 675 CWE entries have the Weakness keyword. These may be children or parents of other entries. There are subdivisions of Base, Class, Variant.
Variant Weakness A weakness that is described at a very low level of detail, typically limited to a specific language or technology.
Base Weakness A weakness that is described in an abstract fashion, but with sufficient details to infer specific methods for detection and prevention. More general than a Variant weakness, but more specific than a Class weakness.
Class Weakness A weakness that is described in a very abstract fashion, typically independent of any specific language or technology.
COMPOUND ELEMENT 9 CWE entries have a COMPOUND ELEMENT keyword. It an Entry that closely associates two or more CWE entries.
CATEGORY 120 CWE entries have a Category keyword. A CWE entry that contains a set of other entries that share a common characteristic. i.e. usually contain lots of children.
VIEW 25 CWE entries have a View keyword. Theses entries are a subset of CWE entries that provides a way of examining CWE content.

Standards are mapped to the most relevant CWE weakness, but may also be relevant to other parent entries. These parent entries are recorded in this documentation. For example: CWE-125 is not mapped to any standards, but is the parent of CWE-126 and CWE-127 which are mapped to standards.
