Compliance Matrix

A compliance matrix for the DERA standards model is available in the Microsoft Excel XML file standards_info/compliance/dera_compliance.xml, which may be found relative to the LDRA tool suite installation directory.

1
Mandatory
All code shall conform to ISO 9899 standard C, with no extensions permitted
Mapped standards
22 S : Use of obsolete language feature ( use = - ).  
110 S : Use of single line comment //.  
632 S : Use of // comment in pre-processor directive or macro defn.  
2
Required
Code written in languages other then C should only be used if there is a defined interface standard for object code to which the compilers/assemblers for both languages conform.
3
Required
Assembly language functions that are called from C should be written as C functions containing only in-line assembly language, and in-line assembly language should not be embedded in normal C code
Mapped standards
17 S : Code insert found.  
88 S : Procedure is not pure assembler.  
4
Mandatory
Provision should be made for appropriate run-time checking
Mapped standards
47 S : Array bound exceeded.  
5
Mandatory
Only those characters and escape sequences which are defined in the ISO C standard shall be used
Mapped standards
113 S : Non standard character in source.   Modifiers : 408 = 0 (Default)
176 S : Non standard escape sequence in source.  
6
Mandatory
Values of character types shall be restricted to a defined and documented subset of ISO 10646-1
7
Mandatory
Trigraphs shall not be used
Mapped standards
81 S : Use of trigraph.  
8
Mandatory
Multibyte characters and wide string literals shall not be used
Mapped standards
82 S : Use of wide string literal.  
9
Mandatory
Comments shall not be nested

A standards message 119 S is issued if /* appears within either a /* or a // style comment.

Mapped standards
119 S : Nested comment found.   Modifiers : 413 = 1, 434 = 0 (Default)
10
Mandatory
Sections of code should not be 'commented out'
Mapped standards
302 S : Comment possibly contains code.  
11
Mandatory
Identifiers (internal and external) shall not rely on significance of more than 31 characters. furthermore the compiler/linker shall be checked to ensure that 31 character significance and case sensitivity are supported for external identifiers
Mapped standards
17 D : Identifier not unique within *** characters.  
61 X : Identifier match in *** chars.  
12
Mandatory
No identifier in one name space shall have the same spelling as an identifier in another name space.
Mapped standards
18 D : Identifier name reused.  
2 S : Label name reused.  
91 S : Name redeclared in another C name space (MR).  
92 S : Duplicate use of a name in an enumeration.  
129 S : Parameter has same name as type or tag.   Modifiers : 191 = 0 (Default)
13
Required
The basic types of char, int, short, long, float, double, unsigned and signed shall not be used, but specific-length equivalents should be typedef'd for the specific compiler, and these type names used in the code
Mapped standards
90 S : Basic type declaration used.   Modifiers : 219 = 0 (Default)
14
Mandatory
The type char shall always be declared as unsigned char or signed char.

89 S does not allow the use of the plain char type and can be disabled if a project allows the use of plain char. The other standards check that objects of plain char type are used in an appropriate way.

Mapped standards
89 S : char type not signed or unsigned.  
329 S : Operation not appropriate to plain char.   Modifiers : 391 = 0 (Default)
431 S : Char used instead of (un)signed char.  
432 S : Inappropriate type - should be plain char.  
15
Required
Floating point implementations should comply with a defined floating point standard
16
Mandatory
The underlying bit representation of floating point numbers shall not be used in any way by the programmer
17
Mandatory
typedef names shall not be reused
Mapped standards
112 S : Typedef name redeclared.   Modifiers : 417 = 0 (Default)
18
Required
Numeric constants should be suffixed to indicate type, where an appropriate suffix is available
Mapped standards
93 S : Value is not of appropriate type.   Modifiers : 191 = 0 (Default), 372 = 0 (Default)
331 S : Literal value requires a U suffix.   Modifiers : 309 = 0 (Default), 358 = 0 (Default)
466 S : long literal with no L suffix.  
19
Mandatory
Octal constants (other than zero) shall not be used
Mapped standards
83 S : Octal number found.  
20
Mandatory
All object and function identifiers shall be declared before use
Mapped standards
19 D : Procedure called before being defined.  
20 D : No declaration for variable found before use.  
21
Mandatory
Identifiers in an inner scope shall not use the same name as an identifier in an outer scope, and therefore hide that identifier
Mapped standards
18 D : Identifier name reused.
