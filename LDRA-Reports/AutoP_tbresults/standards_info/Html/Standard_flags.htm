LDRA Tool Suite Standard Modifiers for C / C++

These standard modifiers may be configured by the user.

Instructions for making alterations can be found in the report.dat file.

107
Number of opens, 0 always open, 1 once only, 2 twice maximum

The default setting is 1.

When the modifier is set to 1, a header file is only opened once, regardless of how many times a header file is included in the source code file being analysed. This can be altered by setting this modifier to one of the following options.

  • 0 Always open include file
  • 1 Each include file opened only once.
  • 2 Each include file opened twice.

Further information can be found in the File Inclusion section of the Testbed User Guide.

110
Asm switch, 0 trailing chars in statement, 1 ; and } separated

The default setting is 1.

When the modifier is set to 1, ; and } are not considered to be part of a preceding ASM statement. Setting the modifier to 0 will treat them as part of the ASM statement.

Further information can be found in the In-Line Assembler Inserts section of the Testbed User Guide.

119
Switch for MISRA standards.

This modifier is set to 1 (in cvals.dat) if you have a MISRA license.

This modifier switches on the standards for:

  • checking prototype and definition parameter lists
  • checking prototype and definition have the same return
  • same number of parameters and same type
  • Actual and formal parameters match
  • Reports illegal chars in include statements.

It also extends checks in the following standards:

361 S requires parenthesis for different operators with the same precedence
120
100 S - Illegal characters in file names.

This modifier lists the characters that are considered by 100 S to be illegal within a filename that is used in a #include statement.

By default the characters are ' \ and ,. \ is banned by some models but may be necessary for the PC. Removing it from the list means that it will not be banned.

122
30 S - Allow ++ and -- as full expression.

The default setting is 1.

When this modifier is set to 1, 30 S is relaxed to allow the use of ++ and in stand-alone expressions and the 3rd expression of a for loop.

127
char - length in bytes.

For example in the vals.dat file:

127    1      char - length in bytes.

This length must match the length in bits in the corresponding tbend.dat file.

     8     width of char

For example in the vals.dat file:

This length must match the length in bits in the corresponding tbend.dat file.

    16     width of unsigned short

This length must match the length in bits in the corresponding tbend.dat file.

    32     width of unsigned int

    64     width of unsigned long

    32     width of float

double - length in bytes.

    64     width of double

136
Permit retention of specified preprocessor macro.

The default setting is 0.

There may be situations where users wish to preserve preprocessor macros. For example: macros may be used in build mechanisms to control whether a procedure is available for unit testing for example.

Such macros may be preserved if this modifier is set to 1, and the relevant macros are included in Section 11 of the c/cppvals.dat file.

11***** Special macro names to convert pre-proc code to comments
         0   6 DUMMY_
         0   8 MYOWNONE
         0  18 EMBED_DECLARATIONS

The number before the name if the number of characters in the name.

Further information on handling macro expansions can be found in the Macro Expansion section of the Testbed User Guide.

149
Add File ID to the to the execution history record.

The default setting is 0.

When analysing a single file, the File ID may be added to the Execution History Record by setting this modifier to 1.

For further information see section The Instrumentation File - cinstr.dat in the Testbed User Guide.

163
201 S, 604 S - Retain macro names.

The default setting is 1.

When this modifier is set to 1, numeric literals substituted via macro names are not included in the checks for 201 S and 604 S.

164
Language Modifier.

This modifier sets the language for the analysis. The possible values are: 

C: 2, C++: 6, JAVA: 13
177
189 S - Maximum source line length.

The default setting is 250.

This modifier set the maximum number of characters in a source line as checked by 189 S.

179
